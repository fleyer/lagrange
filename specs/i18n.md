# Spec: Internationalisation (i18n)

## Intent

Support two languages — French (primary) and English — so pilgrims from all backgrounds can read the site. All user-facing strings live in translation files; components never contain hardcoded copy.

## Package choice: Paraglide JS

**[Paraglide JS](https://inlang.com/m/osslbuzt/paraglide-astro)** (`@inlang/paraglide-astro`) is chosen over alternatives because:

- **Type-safe** — every message key is typed; typos are caught at build time
- **Zero runtime** — compiles translations to static JS, no library shipped to the browser
- **Astro-native** — official Astro adapter, works with static output mode
- **Simple** — no wrapping components or context providers; messages are plain function calls

## Languages

| Code | Language | URL prefix |
|------|----------|------------|
| `fr` | French | `/` (default, no prefix) |
| `en` | English | `/en/` |
| `de` | German | `/de/` |
| `es` | Spanish | `/es/` |

## Setup

1. Install `@inlang/paraglide-astro@0.4.1` as a dev dependency
2. Add the Paraglide Astro integration to `astro.config.mjs`
3. Create `project.inlang/settings.json` with `fr` as source language and `en` as target
4. Create message files:
   - `messages/fr.json` — French strings (source of truth)
   - `messages/en.json` — English strings
5. Import message functions in components: `import * as m from "@/paraglide/messages"`

## Message key conventions

- Keys in `camelCase`, grouped by section: `heroTagline`, `refugeTitle`, `contactEmail`, etc.
- Keys are stable identifiers — never change a key, only the value
- Missing translations fall back to French

## Routing

- Default route `/` serves French
- `/en/` serves English
- `/de/` serves German
- `/es/` serves Spanish
- A language switcher in the `Header` shows all four language codes; the current one is highlighted
